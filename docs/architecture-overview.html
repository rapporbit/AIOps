Multi-Agent AIOps Platform V3 · 架构总览

以 RAG 知识库 + LangGraph 多智能体为核心的智能运维诊断平台 · 七层分层架构

一、分层架构

七层自上而下:接入 → 缓冲 → 编排 → 智能体 → RAG → 工具 → 存储。右侧珊瑚色为各层背靠的外部依赖。

二、诊断请求生命周期

三种入口统一经分布式执行槽调度,按严重度分流到 fast 快速闭环或 deep 深度根因分析,两条链均调用 RAG + MCP + LLM 取证,最终产出报告并审计落库。

三、关键说明

  • 外部依赖:Redis(队列 + 执行槽)、LLM 云(DeepSeek / DashScope)、Milvus + Embedding/Rerank(bge-m3)、5 个 MCP 工具进程(系统 / 网络 / Docker / WebSearch / WinLog)、PostgreSQL 事实库。
  • 并发控制:分布式执行槽用 Redis ZSET + Lua 实现,解决 asyncio.Semaphore 在多进程下失效的问题。
  • 双模式:fast 走 Skill Router→Planner→Executor→Replanner→Report 快速闭环;deep 走多 Agent 并行取证→Evidence Reducer→RCA Judge 深度根因。
  • 检索:Parent-Child 分块 + Vector(Milvus)+ BM25 双路召回 + RRF 融合 + Rerank 精排。
  • 审计:全链路可追溯 task → agent_run → tool_call → evidence,经验沉淀进 Wiki 供下次路由复用。

四、入口点 / 启动

  • 主服务:uvicorn app.main:app — FastAPI + SSE
  • 后台 Worker:python -m app.diagnosis_worker — 消费 Redis 队列
  • 基础设施:docker-compose.yml(Milvus + Redis + Postgres + Attu)
