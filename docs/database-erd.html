关系拓扑

  • 多对多:alerts 与 incident_groups 通过关联表 incident_group_alerts 连接(一条告警可进多个组,一个组聚多条告警)。
  • 一对多:incident_groups 展开 incidents、diagnosis_tasks、evidence。
  • 诊断主链:diagnosis_tasks → agent_runs → tool_calls(任务派生多个 Agent 运行,每个运行调用多个工具)。
  • 无 FK 约束:approval_requests 仅以字段持有 task_id / agent_run_id,无外键约束(图中用虚线表示)。

实现细节

  • 冗余外键:agent_runs / tool_calls 还各自持有 incident_group_id 外键,便于按组聚合查询(图中省略)。
  • 删除行为:挂在 incident_groups 下的子表多为 ON DELETE CASCADE(删组级联清理);而 evidence.incident_id、agent_runs.task_id 等用 ON DELETE SET NULL(子记录可独立存活)。
  • 幂等去重:diagnosis_tasks 用 dedup_key + 部分唯一索引(仅 pending/running 时生效)实现入队去重。
  • 类型约定:主键/外键均为 TEXT;标签、注解、payload、元数据等用 JSONB;时间戳统一 TIMESTAMPTZ。
